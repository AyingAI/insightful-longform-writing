# Prompt Examples

这些示例用于触发并使用 `insightful-longform-writing` Skill。可以直接复制，也可以按你的主题替换关键词。

## 从零开始写文章

```text
请用 insightful-longform-writing 的方法，帮我写一篇 3000-4000 字的中文长文。
主题：为什么团队需要把 AI 使用经验沉淀成 Skill
读者：已经用过 AI 工具，但还没有系统方法的产品、运营和研发同学
目标：让他们意识到 Skill 不是“提示词收藏夹”，而是团队经验资产
要求：先给写作定位和大纲，不要直接写全文
```

```text
我想写一篇科普文章，主题是“AI Agent 的记忆机制到底在解决什么问题”。
读者听说过 Agent，但不理解 memory、context、retrieval 的区别。
文章希望 10 分钟读完。
请先给我文章主线、章节结构和每章关键案例。
```

## 只要大纲

```text
先不要写正文。请用这个 Skill 的方法，为“为什么个人知识管理工具总是用不起来”设计一篇公众号长文大纲。
要求：
- 开头要有具体场景
- 每章标题要有问题感
- 每章说明核心观点、案例和下一章承接
```

```text
请给我一篇行业分析文章的大纲。
主题：企业为什么开始从“买 SaaS”转向“搭 Agent 工作流”
读者：企业数字化负责人
目标：帮他们理解变化背后的驱动力和判断变量
```

## 逐章写草稿

```text
大纲我确认了。先写引子和第一章。
要求：
- 第一章不要有标题
- 前 3 句话内点题
- 用一个具体人物和具体任务开头
- 不要写成“核心观点”式开场
```

```text
继续写第二章。
请承接上一章，不要硬切换。
这一章重点解释“为什么工具本身不能沉淀经验”。
需要一个贴近日常工作的类比，但不要堆多个比喻。
```

## 修订已有文章

```text
请按 insightful-longform-writing 的标准，帮我审阅下面这篇文章。
重点看：
1. 开头是否在前 3 句话内点题
2. 章节之间是否递进
3. 是否有太多 bullet list
4. 术语解释是否足够具体
5. 结尾是否有洞察和行动钩子
```

```text
请只改这篇文章的开头。
目标：更快点题、更有具体场景、更少空泛铺垫。
不要改正文其他部分。
```

```text
这章读起来像报告，请帮我改成更自然的长文叙述。
要求：
- 去掉“价值 1 / 价值 2 / 价值 3”的并列感
- 保留信息密度
- 用承接句把段落串起来
- 结尾落到一个洞察，而不是写“启发”
```

## 处理术语和概念

```text
请帮我重写这段概念解释。
术语：渐进式披露（Progressive Disclosure）
读者：产品经理和运营同学
要求：保留中英术语，用一句话解释，再接一个生活化类比和一个 AI 使用场景。
```

```text
这篇文章里有很多专业词。请帮我判断哪些术语应该保留并解释，哪些可以换成日常表达。
判断标准：这个词未来是否会反复出现，读者懂了是否能理解更多内容。
```

## 不同题材适配

```text
请把这个选题写成“产品解读”而不是科普文。
主题：某个 AI 写作工具的新功能为什么重要
重点：使用场景、旧问题、新机制、影响边界
先给大纲。
```

```text
请把这个选题写成“方法论文章”。
主题：如何把一次成功的 AI 协作流程沉淀成团队 Skill
要求：读者看完能照着做，但不要写成干巴巴的操作手册。
```

```text
请把这个选题写成“观点评论”。
主题：为什么我不认为 Agent 会马上替代所有白领工作
要求：有清晰立场，先回应常见误读，再给出新的解释框架。
```

## 全文质检

```text
全文已经写完。请按 Skill 的通读清单做一次质检。
请输出：
- 最大的 3 个结构问题
- 最需要加强的 3 个案例或解释
- 可以删除或合并的段落
- 标题和结尾的优化建议
```

```text
请检查这篇文章有没有以下问题：
- 第一人称过密
- 章节结尾模板化
- 同一个概念重复解释
- 数据或案例没有出处感
- 金句太像口号，不像判断
```

## 插图规划

```text
全文已经定稿。请用 Skill 的插图规划方法，帮我规划 3-4 张配图。
每张图请包含：位置、用途、核心信息、类比对象、画面结构、生图 prompt 和图注。
默认风格：扁平化、企业科技风、专业但不幼稚。
```

```text
请不要生成图片，只写插图 prompt。
文章主题：Skill 如何把个人经验变成团队资产
要求：每张图都必须有信息量，不要装饰性配图。
```

## 一次性完整稿

```text
请用 insightful-longform-writing 的方法，直接写一篇完整稿。
主题：为什么 AI 工具越多，团队越需要统一工作流
读者：业务团队负责人
篇幅：约 2500 字
要求：虽然直接写完整稿，但请遵守这个 Skill 的结构、术语解释、案例和结尾标准。
```

不推荐一开始就这样使用。更好的方式是先大纲、再逐章写作。
